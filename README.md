1. О проекте
Назначение:
Веб-приложение для мониторинга и анализа приёмной кампании университета в реальном времени. Система позволяет отслеживать динамику подачи заявлений, процесс зачисления и конверсию с возможностью детальной фильтрации по направлениям подготовки, источникам трафика и волнам зачисления.

Ключевая задача
Обеспечить приёмную комиссию, деканаты факультетов и аналитический отдел инструментом для быстрого выявления проблемных мест в наборе студентов и оперативного реагирования на изменения ситуации.

Основные возможности
📊 Интерактивный дашборд с ключевыми метриками

🔍 Детальная фильтрация по периодам, программам, источникам, волнам

📈 Визуализация динамики заявок и конверсии

👤 Просмотр и управление карточками заявок

📥 Экспорт отчётов в CSV/Excel

🔄 Интеграция с внешними системами через API

👥 Ролевой доступ (приёмная комиссия, факультеты, аналитики)

2. Бизнес-цели и KPI
Бизнес-цели
Цель	Метрика успеха
Повысить скорость реакции на провалы набора	Время обнаружения проблемы < 1 дня
Увеличить конверсию "заявка → зачисление"	Рост конверсии на 5% за кампанию
Оптимизировать работу с источниками трафика	Выявление топ-3 эффективных источников
Сократить время обработки заявок	Снижение среднего времени на 20%
KPI (Ключевые показатели эффективности)
Метрика	Формула	Целевое значение
Конверсия	enrolled / applications * 100%	Аналитика по срезам
Время отклика дашборда	Время перестроения при смене фильтров	≤ 2 секунды
Количество заявок	COUNT(applications)	Мониторинг динамики
Количество зачисленных	COUNT(applications WHERE status = 'enrolled')	Мониторинг динамики
Среднее время обработки	AVG(status_changed_at - created_at)	Мониторинг эффективности
Отслеживаемые метрики на дашборде
✅ Applications — общее количество заявок

✅ Enrolled — количество зачисленных

✅ Conversion — конверсия в %

✅ Avg Processing Time — среднее время обработки заявки

✅ Conversion by Source — конверсия по источникам трафика

✅ Conversion by Wave — конверсия по волнам зачисления
